    "# Data organisation\n",
    "IBL data is structured using the ALyx File (ALF) name specification.\n",
    "Please see this [section](../../alf.html) for a detailed introduction to the ALF format\n",
    "(especially the sections on Collections, Revisions and Namespaces).\n",
    "\n",
    "Here we further describe some key ALF concepts in the context of the IBL data.\n",
    "\n",
    "## Session folder\n",
    "As per the ALF convention, each experiment has a unique session folder which has the following pattern,\n",
    "```\n",
    "lab/Subjects/subject/date/session_number\n",
    "e.g cortexlab/Subjects/KS023/2020-03-04/001\n",
    "```\n",
    "\n",
    "## Datasets and data file structure\n",
    "The following file structure shows a subset of the datasets used in IBL.\n",
    "For a full description of all datasets please see this [document](https://docs.google.com/document/d/1OqIqqakPakHXRAwceYLwFY9gOrm8_P62XIfCTnHwstg/edit).\n",
    "\n",
    "    lab/\n",
    "    ├─ Subjects/\n",
    "    |  ├─ subject/\n",
    "    |  |  ├─ 2021-06-30/\n",
    "    |  |  │  ├─ 001/\n",
    "    |  |  │  │  ├─ alf/\n",
    "    |  |  │  │  │  ├─ probe00/\n",
    "    |  |  │  │  │  │  ├─ spikes.clusters.npy\n",
    "    |  |  │  │  │  │  ├─ spikes.times.npy\n",
    "    |  |  │  │  │  ├─ probe01/\n",
    "    |  |  │  │  │  │  ├─ #2021-07-05#/\n",
    "    |  |  │  │  │  │  │  ├─ spikes.clusters.npy\n",
    "    |  |  │  │  │  │  │  ├─ spikes.times.npy\n",
    "    |  |  │  │  │  │  ├─ spikes.clusters.npy\n",
    "    |  |  │  │  │  │  ├─ spikes.times.npy\n",
    "    |  |  │  │  |  ├─ _ibl_trials.intervals.npy\n",
    "    |  |  │  │  |  ├─ probes.description.json\n",
    "\n",
    "As per the [ALF](../../alf.html) convention, a few things should be noticed:\n",
    "- spikes collected on two different probes are grouped into two separate collection folders:\n",
    "```\n",
    "alf/probe00\n",
    "alf/probe01\n",
    "```\n",
    "- the usage of the namespace `_ibl_` in `_ibl_trials.intervals.npy` indicates that\n",
    "this dataset is specific to the IBL and not a standard in the broad scientific community.\n",
    "- the usage of a revision folder for storing a new spike sorting output:\n",
    "```\n",
    "alf/probe01/#2021-07-05#/spikes.clusters.npy\n",
    "alf/probe01/spikes.clusters.npy\n",
    "```\n",
    "\n",
    "## Experiment IDs\n",
    "Each experimental session is identified by a unique identifier known as an experiment ID (eid). The most common way of\n",
    "representing eids within the IBL are,\n",
    "\n",
    "```\n",
    "(str) : An experiment UUID as a string\n",
    "e.g  '\n",
    "(str) : A session string of the form <subject>/<date>/<number>\n",
    "e.g 'KS023/2019-12-10/001'\n",
    "(Path) : A pathlib ALF path of the form <lab>/Subjects/<subject>/<date>/<number>\n",
    "e.g Path(r'cortexlab/Subjects/KS023/2019-12-10/001')\n",
    "```\n",
    "\n",
    "This unique identifier is extremely important, as it enables scientists to search for and download the data associated with\n",
    "a specific experimental session stored in the database.\n",
    "\n",
    "For more information on data search and download, please visit the [ONE](../../one.html) section.\n",
    "Note: ONE also supports a range of other experimental session representations. Please see this\n",
    "[tutorial](../../notebooks_external/docs_experiment_ids.html) for more details on session representation\n",
    "and for useful conversion tricks."
